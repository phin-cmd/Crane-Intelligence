## Color Palette

### Primary Colors

#### Dark Backgrounds
- **Primary Black**: `#0F0F0F` - Main email background
- **Secondary Black**: `#1A1A1A` - Content sections
- **Tertiary Black**: `#121212` - Footer and header gradients

#### Accent Colors
- **Accent Green**: `#00FF85` - Primary CTA buttons, links, highlights
- **Accent Yellow**: `#FFD600` - Secondary CTAs, warnings, special offers
- **Accent Red**: `#FF4444`, `#FF5330` - Alerts, errors, urgent actions

#### Text Colors
- **Primary Text**: `#FFFFFF` - Headings, important content
- **Secondary Text**: `#B0B0B0` - Body text, descriptions
- **Muted Text**: `#808080` - Footer text, disclaimers, timestamps

#### Border Colors
- **Border Dark**: `#333333` - Primary borders, dividers
- **Border Light**: `#404040` - Subtle borders, separators

### Color Usage Guidelines

**Primary CTAs (Green #00FF85):**
- Use for main call-to-action buttons
- "Get Started", "Access Dashboard", "Order Report"
- Links to primary actions
- Success indicators

**Secondary CTAs (Yellow #FFD600):**
- Use for secondary actions
- "Learn More", "View Details", "Add to Calendar"
- Special offers and promotions
- Warning messages

**Red (#FF4444, #FF5330):**
- Use sparingly for alerts and urgent actions
- Error messages
- Critical notifications
- Delete/remove actions

**Accessibility:**
- All color combinations meet WCAG AA standards
- Text on dark backgrounds maintains 4.5:1 contrast ratio minimum
- Green buttons (#00FF85) on dark backgrounds (#0F0F0F) = 4.8:1 contrast ratio
- Yellow buttons (#FFD600) on dark backgrounds = 5.2:1 contrast ratio

## Typography

### Font Families

**Headings:**
- Primary: `'Roboto Condensed'`
- Fallback: `'Inter', Arial, sans-serif`
- Weight: 600-700 (Semi-bold to Bold)
- Letter-spacing: Optimized for readability (0.5px to 1px)

**Body Text:**
- Primary: `'Inter'`
- Fallback: `-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, sans-serif`
- Weight: 400-500 (Regular to Medium)
- Base size: 16px

**Monospace (Technical Content):**
- Primary: `'JetBrains Mono'`
- Fallback: `'Fira Code', 'Consolas', monospace`
- Use for: Report IDs, codes, technical specifications

### Typography Hierarchy

**H1 (Main Headline):**
- Font: Roboto Condensed
- Size: 28px (desktop), 24px (mobile)
- Weight: 700
- Color: #00FF85 (accent green) or #FFFFFF
- Line-height: 1.2
- Margin: 0 0 20px 0

**H2 (Section Headers):**
- Font: Roboto Condensed
- Size: 24px (desktop), 20px (mobile)
- Weight: 600
- Color: #00FF85 (accent green) or #FFFFFF
- Line-height: 1.3
- Margin: 0 0 16px 0

**H3 (Subsection Headers):**
- Font: Roboto Condensed
- Size: 20px (desktop), 18px (mobile)
- Weight: 600
- Color: #FFFFFF
- Line-height: 1.4
- Margin: 0 0 12px 0

**Body Paragraph:**
- Font: Inter
- Size: 16px (desktop), 14px (mobile)
- Weight: 400
- Color: #B0B0B0
- Line-height: 1.6
- Margin: 16px 0

**Small Text (Footer, Disclaimers):**
- Font: Inter
- Size: 12px
- Weight: 400
- Color: #808080
- Line-height: 1.6

## Logo Usage

### Logo Specifications

**Standard Sizes:**
- Desktop: 200px width, 40-60px height
- Mobile: 150-180px width, 35-50px height
- Maximum width: 200px to maintain readability

**Placement:**
- Centered in email header
- Minimum 20px margin below logo
- Minimum 30px padding on all sides
- Always above the fold (visible without scrolling)

### Logo Implementation

**Primary Method: Inline SVG**
- Use inline SVG for maximum email client compatibility
- Includes fallback PNG reference
- Optimized for email rendering
- No external dependencies

**Fallback Method: Hosted Image**
- URL: `https://craneintelligence.tech/images/logos/crane-intelligence-logo-white.svg`
- PNG fallback: `https://craneintelligence.tech/images/logos/crane-intelligence-logo.png`
- Always include alt text: "Crane Intelligence"

**Logo Variations:**
- White version for dark backgrounds (default)
- Maintain consistent sizing across all emails
- Never stretch or distort the logo
- Maintain aspect ratio (approximately 5:1 width to height)

### Logo Do's and Don'ts

**Do:**
- ✅ Use the logo in the header of every email
- ✅ Center-align the logo
- ✅ Use white/light version on dark backgrounds
- ✅ Maintain proper spacing around the logo
- ✅ Include alt text for accessibility

**Don't:**
- ❌ Never modify logo colors
- ❌ Never stretch or distort the logo
- ❌ Never place logo in footer (header only)
- ❌ Never use logo smaller than 150px width
- ❌ Never remove or replace logo elements

## Email Client Compatibility

### Tested Email Clients

**Desktop Clients:**
- Gmail (web, Chrome, Firefox)
- Outlook 2016, 2019, 365
- Apple Mail (macOS)
- Yahoo Mail (web)

**Mobile Clients:**
- Gmail app (iOS, Android)
- Apple Mail (iOS)
- Outlook app (iOS, Android)
- Native email clients

### Compatibility Best Practices

**Layout:**
- Use table-based layouts for maximum compatibility
- Avoid CSS Grid and Flexbox (limited support)
- Use inline styles for critical formatting
- Test in Outlook (most restrictive client)

**CSS:**
- Inline critical styles
- Use `<style>` tag in `<head>` for non-critical styles
- Avoid advanced CSS features (animations, transforms)
- Use media queries for responsive design

**Fonts:**
- Always include fallback fonts
- Web fonts may not load in all clients
- Test with fallback fonts only
- Use system fonts as fallbacks

**Images:**
- Always include alt text
- Use width and height attributes
- Provide fallback text for images
- Optimize images for email (smaller file sizes)

**Responsive Design:**
- Breakpoint at 600px (standard email width)
- Breakpoint at 480px (mobile devices)
- Stack content vertically on mobile
- Increase touch target sizes (minimum 44x44px)

## Best Practices

### Content Strategy

**Single Primary CTA:**
- One main action per email
- Clear, action-oriented button text
- Prominent placement (above the fold when possible)

**Scannable Content:**
- Use clear headings and subheadings
- Break content into short paragraphs
- Use bullet points and lists
- White space for readability

**Personalization:**
- Use recipient's name when available
- Reference previous interactions
- Tailor content to user segment
- Dynamic content based on user data

**A/B Testing:**
- Test subject lines
- Test CTA button colors and text
- Test email layouts
- Test send times
- Track open rates, click rates, conversions

### Accessibility Guidelines

**Alt Text:**
- All images must have descriptive alt text
- Decorative images: use empty alt="" attribute
- Informative images: describe the content

**Color Contrast:**
- Minimum 4.5:1 contrast ratio for normal text
- Minimum 3:1 contrast ratio for large text (18px+)
- Test with color blindness simulators
- Don't rely on color alone to convey information

**Readable Font Sizes:**
- Minimum 14px for body text
- Minimum 16px recommended
- Larger fonts for mobile (16px+)

**Semantic HTML:**
- Use proper heading hierarchy (h1, h2, h3)
- Use lists for list content
- Use tables for tabular data
- Proper link text (not "click here")

**Keyboard Navigation:**
- All interactive elements should be keyboard accessible
- Clear focus indicators
- Logical tab order

### Performance

**Image Optimization:**
- Compress images before embedding
- Use appropriate formats (PNG for logos, JPG for photos)
- Maximum image width: 600px
- Lazy loading not recommended (many clients don't support)

**File Size:**
- Keep HTML under 100KB when possible
- Minimize inline CSS
- Remove unnecessary code
- Test loading times

**Loading:**
- Ensure email renders quickly
- Test with slow connections
- Provide text fallbacks for images

## Compliance

### CAN-SPAM Act (US)

**Requirements:**
- Include physical mailing address
- Provide clear unsubscribe mechanism
- Honor unsubscribe requests within 10 business days
- Identify message as advertisement (if applicable)
- Include accurate sender information

### GDPR (EU)

**Requirements:**
- Obtain explicit consent for marketing emails
- Provide clear privacy policy
- Easy unsubscribe process
- Right to be forgotten
- Data processing transparency

### Best Practices for All Regions

- Always get permission before sending
- Make unsubscribe easy and clear
- Respect user preferences
- Provide clear sender identification
- Include contact information
- Honor opt-out requests promptly
